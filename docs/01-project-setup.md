# Epic 01: Project Setup

Wails v2 project scaffolding, directory structure, Go module configuration, platform-specific data paths, and development workflow.

## 1. Wails v2 Project Initialization

### Prerequisites Check

```bash
# Install Wails CLI
go install github.com/wailsapp/wails/v2/cmd/wails@latest

# Verify environment
wails doctor
```

Required:
- Go 1.25+ (binary at `/Users/stefan.balea/sdk/go1.25.3/bin/go`)
- Node.js 20+ with npm
- Platform toolchains (see below)

### Platform-Specific Dependencies

| Platform | Required |
|----------|----------|
| macOS | Xcode Command Line Tools (`xcode-select --install`) |
| Linux | `libgtk-3-dev`, `libwebkit2gtk-4.0-dev`, `build-essential` |
| Windows | WebView2 Runtime (pre-installed on Windows 10/11), NSIS for installer builds |

### Project Scaffold

```bash
wails init -n bze-hub -t react-ts
```

This creates the base structure which we'll customize as described below.

## 6. Data Directories

### Platform-Specific Paths

```go
func AppDataDir() string {
    switch runtime.GOOS {
    case "darwin":
        return filepath.Join(os.Getenv("HOME"), "Library", "Application Support", "bze-hub")
    case "windows":
        return filepath.Join(os.Getenv("APPDATA"), "bze-hub")
    case "linux":
        // XDG_DATA_HOME or fallback
        if xdg := os.Getenv("XDG_DATA_HOME"); xdg != "" {
            return filepath.Join(xdg, "bze-hub")
        }
        return filepath.Join(os.Getenv("HOME"), ".local", "share", "bze-hub")
    }
}
```

### Subdirectory Layout


## 7. Main Entry Point (main.go)

```go
package main

import (
    "embed"
    "github.com/wailsapp/wails/v2"
    "github.com/wailsapp/wails/v2/pkg/options"
    "github.com/wailsapp/wails/v2/pkg/options/assetserver"
)

//go:embed all:frontend/dist
var assets embed.FS

func main() {
    app := NewApp()

    err := wails.Run(&options.App{
        Title:     "BZE Hub",
        Width:     1280,
        Height:    800,
        MinWidth:  800,
        MinHeight: 600,
        AssetServer: &assetserver.Options{
            Assets: assets,
        },
        OnStartup:  app.startup,
        OnShutdown: app.shutdown,
        Bind: []interface{}{
            app,
        },
    })
    if err != nil {
        panic(err)
    }
}
```

## 9. Development Workflow

### Daily Development

```bash
# Start development mode (hot reload for frontend, auto-rebuild for Go)
cd bze-hub
wails dev
```

This opens the app with browser DevTools available for debugging the React shell.

### Testing

```bash
# Go backend tests
go test ./internal/...

# Frontend tests
cd frontend && npm test

# Manual testing checklist:
# 1. App launches without errors
# 2. Dashboard tab renders
# 3. dApp iframes load (requires internet)
# 4. Keplr bridge injection works (check browser console in iframe)
# 5. Node download and init completes
# 6. Wallet creation and signing flow works
```

### Building

```bash
# Local platform build
wails build

# Specific platform (for cross-compilation)
wails build -platform darwin/arm64
wails build -platform windows/amd64
wails build -platform linux/amd64
```

## 10. Wails Events

Wails supports events for Go-to-frontend communication (push model):

```go
// Go side - emit events
runtime.EventsEmit(a.ctx, "node:status-changed", status)
runtime.EventsEmit(a.ctx, "wallet:account-changed", address)
runtime.EventsEmit(a.ctx, "bridge:sign-request", request)
```

```typescript
// Frontend side - listen for events
import { EventsOn } from '../wailsjs/runtime/runtime';

EventsOn("node:status-changed", (status) => {
  setNodeStatus(status);
});

EventsOn("bridge:sign-request", (request) => {
  showApprovalDialog(request);
});
```

This avoids polling for node status and enables real-time UI updates.
